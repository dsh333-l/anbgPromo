## 五、施工注意事项

> 标准提醒，固定文案，不随项目变化。

1. 所有洞口尺寸为现场测量结果，正式下单前需复测一次。
2. 开启方向以本方案标注为准，安装前请与施工方确认开启方向是否影响家具摆放。
3. 框体颜色以实物色卡为准，本文档效果图仅供参考，可能存在色差。
4. 分格比例和尺寸为设计示意，实际生产以工厂深化图纸为准。
5. 玻璃配置可根据楼层、朝向、隔音要求二次调整。

## 变量替换映射表

导出时按以下映射从项目状态文件提取数据：

| 模板变量 | 数据来源 |
|---------|---------|
| `{{客户姓名}}` | `项目状态.md` 客户姓名段 |
| `{{项目名称}}` | `项目状态.md` 项目名称段 |
| `{{设计师署名}}` | `项目状态.md` 品牌署名段 |
| `{{出图日期}}` | 当天日期 |
| `{{方案版本号}}` | 默认 1.0；客户要求修改后递增 |
| `{{总楼层}}` `{{总洞口数}}` | 遍历 `*-最终方案.md` 自动统计 |
| `{{整屋风格}}` | `项目状态.md` 整屋风格段 |
| `{{主推产品系列}}` | 遍历 `*-最终方案.md` 出现频次最高 |
| `{{主框体颜色}}` | 遍历 `*-最终方案.md` 出现频次最高 |
| `{{房间名}} {{洞口编号}} {{尺寸}} ...` | 各 `*-最终方案.md` 内的元数据段 |
| `{{分格图路径}}` | `分格效果图/[房间标签]/[洞口编号]/最终版/` 内的图片 |
| `{{整屋图路径}}` | `整屋效果图/[房间标签]/[洞口编号]/最终版/` 内的图片 |
| 统计表 | 遍历 `*-最终方案.md` 后聚合计数 |

## 导出后的可选格式

Markdown 文件生成后，如客户需要其他格式：

```bash
# 导出 PDF（需安装 pandoc 和 wkhtmltopdf）
pandoc proposal.md -o proposal.pdf --pdf-engine=wkhtmltopdf

# 导出 Word
pandoc proposal.md -o proposal.docx
```

注意：图片路径需要先转为绝对路径或与 Markdown 同目录。

## 更新本模板

如果品牌方案模板需要调整（增删字段、修改文案、更换 logo 位置），直接编辑本文件的"模板正文"段，下次导出会自动使用最新版本。
