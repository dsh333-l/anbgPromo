# 即梦 CLI 设计出图工作流 (dreamina-design-workflow.md)

> 版本：v1.0 | 用途：门窗设计师调用即梦 CLI 完成分格效果图和整屋效果图。
> 与视频制作的 `dreamina-cli-workflow.md` 保持一致基础规则，但补充设计场景的 prompt 模板和参数。

---

## Agent 总原则

- 所有图像生成统一使用 `dreamina` CLI，不使用其他工具。
- 提交任务前，先确认 CLI 已安装、已登录、`user_credit` 正常。
- **传给即梦的 prompt 必须是 7 层结构整段，不允许压缩成单段描述**。
- 设计场景下 7 层结构的【运镜节奏】层改为【视角构图】，【人物/产品设定】层改为【门窗规格】，【分镜脚本】层改为【分格设计】或【室内陈设】。
- 默认使用 `--poll=30`；超时后用 `query_result` 拉结果。
- 凡是基于客户上传的洞口照片出图，**必须用 `image2image`**，不要用 `text2image`，否则丢失现场透视。

## 安装与登录

> 首次使用时执行；如果已经在视频制作或其他 skill 里登录过，直接跳过。

```bash

# 自检

# 只在 user_credit 失败时才执行

```

## 设计出图常用命令

### 1. 分格效果图（基于洞口照片）

```bash
dreamina image2image \
  --images ./output/设计方案/[客户-项目名]/洞口照片/[房间标签]/[洞口编号]/照片/[原图] \
  --prompt="$(cat prompt-window-7layer.txt)" \
  --resolution_type=2k \
  --poll=30
```

适用：

- 客户已上传现场洞口照片
- 需要保留洞口的真实透视、墙面、室外环境

### 2. 整屋效果图（基于分格图）

```bash
dreamina image2image \
  --images ./output/设计方案/[客户-项目名]/分格效果图/[房间标签]/[洞口编号]/第1版/[分格图] \
  --prompt="$(cat prompt-room-7layer.txt)" \
  --resolution_type=2k \
  --poll=30
```

适用：

- 已有分格效果图
- 需要在保留窗的基础上延展室内陈设

### 3. 概念图（无照片场景）

```bash
dreamina text2image \
  --prompt="$(cat prompt-concept-7layer.txt)" \
  --ratio=4:3 \
  --resolution_type=2k \
  --poll=30
```

适用：

- 客户暂无现场照片
- 用于风格预览或方案沟通

### 4. 方案封面图

```bash
dreamina text2image \
  --prompt="$(cat prompt-cover-7layer.txt)" \
  --ratio=16:9 \
  --resolution_type=2k \
  --poll=30
```

适用：

- 导出方案文档时的封面
- 整屋全景或品牌氛围图

## 7 层结构 Prompt 模板（设计版）

### 模板 A：分格效果图

```text
【风格锚定】[设计调性，例如"建筑级渲染图，写实摄影质感，细节级精度"]
【光影色调】[光线来源 + 色温 + 画质，例如"自然日光从窗外射入，柔和均匀，画面通透干净，4K超清"]
【视角构图】[拍摄角度，例如"室内正对洞口的水平视角，能完整看到整个窗体和两侧墙体"]
【门窗规格】
  系列：[产品系列名]
  开启方式：[例如 三联固定+两扇内开内倒]
  框体颜色：[色号或描述]
  玻璃：[玻璃配置]
  边框宽度：[窄/标准/宽]
【分格设计】
  [详细分格描述，例如：横向 3 联，左右两扇为内开内倒，中间为大固定，分格比例 1:2:1，水平横档位于离地 1500mm 处]
【场景环境】[房间类型 + 楼层 + 朝向 + 室外可见元素，例如"高层主卧南向，窗外远景为城市天际线"]
【整体氛围】[一句话定调，例如"克制专业，写实自然，无营销修饰"]
```

### 模板 B：整屋效果图

```text
【风格锚定】[整屋风格名 + 渲染调性，例如"现代简约室内空间，建筑级渲染图，写实摄影质感"]
【光影色调】[光线来源 + 色温 + 画质]
【视角构图】[室内构图角度，例如"室内正对窗的水平构图，能完整看到窗、墙、地面、吊顶"]
【门窗规格】[沿用模板 A 的门窗规格段]
【室内陈设】
  吊顶：[预设吊顶]
  地面：[预设地面]
  墙面：[预设墙面]
  窗套：[预设窗套]
  窗台板：[预设窗台板]
  窗帘：[预设窗帘]
  家具：[预设家具大件]
【场景环境】[房间类型 + 楼层 + 朝向]
【整体氛围】[一句话定调]
```

### 模板 C：方案封面图

```text
【风格锚定】[品牌氛围图，例如"安贝格门窗品牌氛围图，建筑级渲染，电影级光影"]
【光影色调】[黄昏暖光 / 清晨自然光 / 室内暖光]
【视角构图】[宽幅 16:9，室内全景或建筑外立面]
【门窗规格】[本项目主推产品系列和颜色]
【画面构图】
  主体：[落地大窗 / 阳台门 / 客厅全景]
  前景：[简洁家具]
  中景：[窗与窗外景色交融]
  远景：[城市/自然]
【场景环境】[整屋风格定调]
【整体氛围】[一句话品牌调性，例如"安静、克制、有质感的居住感"]
```

## 调用方式

> 7 层结构 prompt 内容多、含特殊符号【】，**强烈建议先存到文件再 cat 注入**。

```bash
# 把 7 层 prompt 写入文件
cat > /tmp/prompt-7layer.txt <<'EOF'
【风格锚定】...
【光影色调】...
【视角构图】...
【门窗规格】...
【分格设计】...
【场景环境】...
【整体氛围】...
EOF

# 用 cat 注入
dreamina image2image \
  --images ./洞口照片/F2-主卧-南/洞口01/照片/原图.jpg \
  --prompt="$(cat /tmp/prompt-7layer.txt)" \
  --resolution_type=2k \
  --poll=30
```

或用 here-doc 直接传入：

```bash
dreamina image2image \
  --images ./洞口照片/F2-主卧-南/洞口01/照片/原图.jpg \
  --prompt="$(cat <<'EOF'
【风格锚定】...
```

## 异步任务处理

### 使用 `--poll`

默认使用 `--poll=30`。

### 查询结果

```bash
dreamina query_result --submit_id=<你的 submit_id>
```

### 直接下载到目录

```bash
dreamina query_result --submit_id=<你的 submit_id> --download_dir=./output/设计方案/[客户-项目名]/分格效果图/[房间标签]/[洞口编号]/第1版/
```

### 查看历史任务

```bash
dreamina list_task
dreamina list_task --gen_status=success
```

## 命令选择规则（设计场景）

| 场景 | 命令 |
|------|------|
| 客户上传了洞口照片 → 分格效果图 | `image2image` |
| 已有分格图 → 整屋效果图 | `image2image` |
| 客户上传了手绘草图 → 风格化渲染 | `image2image` |
| 没有任何图片 → 概念图/封面图 | `text2image` |
| 客户对某张图想做小幅风格调整 | `image2image`（指向那张图） |
| 查询历史任务 | `list_task` / `query_result` |

## 结果落盘

| 文件类型 | 路径 |
|---------|------|
| 分格效果图 | `output/设计方案/[客户-项目名]/分格效果图/[房间标签]/[洞口编号]/第[N]版/` |
| 整屋效果图 | `output/设计方案/[客户-项目名]/整屋效果图/[房间标签]/[洞口编号]/第[N]版/` |
| 方案封面图 | `output/设计方案/[客户-项目名]/成品方案/封面/` |
| 任务 JSON 和 submit_id | 与图片同目录的 `task.json` |

每次迭代生成新版本号（`第1版` `第2版` `第3版`...），不覆盖旧文件。`最终版/` 目录用于客户最终确认的版本。

## 常见问题

### 出图丢失了原洞口的形状或透视

- 检查是否误用了 `text2image`，应改为 `image2image`
- prompt 里加强【视角构图】描述，明确"保持原始洞口比例和透视"
- 必要时降低风格强度，让结构主导

### 整屋效果图里窗户被改了颜色或分格

- 在 prompt【门窗规格】层强调"沿用输入图中的窗体结构和颜色"
- 把【分格设计】层完整复制过来，避免模型自由发挥

### 框体颜色不准

- 在【门窗规格】层用更具体的描述，例如"matte pure black aluminum, no reflection, deep matte finish"
- 颜色只用标准色名，避免使用色号（模型不认识 C01）

### 整屋家具风格跑偏

- 在【室内陈设】层每一项都用具体名词，避免抽象词
- 例如不要写"有质感的沙发"，要写"米色亚麻三人位沙发"
