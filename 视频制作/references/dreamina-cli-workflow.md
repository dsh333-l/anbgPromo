# 即梦 CLI 操作流程 (dreamina-cli-workflow.md)

> 版本：v1.2
> 用途：视频制作读取此文件，统一通过即梦 CLI 完成图片和视频生成。
> 方法来源：用户提供的《即梦 CLI 体验指南》。
> 重要说明：CLI 品牌名是即梦，但命令名统一使用 `dreamina`。

---

## Agent 总原则

- 所有视频生成任务统一使用 `dreamina` CLI，不再使用网页工具或 Kling。
- **视频生成默认命令为 `multimodal2video`**（全能参考模式，支持图+视频+音频混合输入）。除非用户明确要求其他命令模式，所有视频任务都走 `multimodal2video`。
- **视频生成默认模型为 `seedance 2.0 fast vip`**，通过 `--model_version seedance2.0fast_vip` 指定。除非用户明确要求其他模型，所有视频命令必须带此参数。
- **参考图统一约定**：
  - **图片1 = 人物参考（默认必带）**：每条视频都需要，用于锁定门窗顾问/角色形象的一致性。`--image` 的第 1 张永远是人物参考。
  - **图片2 = 产品参考（按需带）**：只在内容**明确介绍产品优点**（结构、工艺、卖点、对比、测评等）时才追加。纯情绪钩子、纯生活场景、纯科普概念等不涉及具体产品展示的视频，**不要带图片2**。当带图片2时，它是 `--image` 的第 2 张。
  - Prompt 里要把"图片1""图片2"作为内联标记直接写在【人物/产品设定】或对应分镜的画面描述里，让模型知道每张参考图绑定的角色。**不带图片2 的视频，prompt 里也不要出现"图片2"标记**。
- 提交任务前，先确认 CLI 已安装、已登录、余额查询正常。
- 默认复用现有登录态，不要每次任务都重新执行 `dreamina login`。
- 默认优先使用 `--poll=30`，先等待一轮结果；超时后再用 `query_result` 追结果。
- **每次提交生成任务后，立即用 `open` 命令打开即梦网站**，让用户可以在浏览器中查看任务进度和结果预览。命令：`open "https://jimeng.jianying.com"` 。
- 如果任务依赖本地图片，必须先确认文件路径真实存在，再执行命令。
- 如果文档方法和其他旧说明冲突，以本文件为准。

## 登录与自检

### 登录

```bash
dreamina login
```

- 该命令会自动拉起默认浏览器完成授权。
- 如果当前机器没有浏览器，CLI 会引导你手动导入登录态。
- 这不是每次都要执行的命令。只有首次登录、账号切换、登录态失效时才需要。

### 登录排障

```bash
dreamina login --debug
```

- 当浏览器没有自动拉起，或登录卡住时使用。
- 终端会打印回调地址和调试信息，便于排查。

### 登录成功自检

```bash
dreamina user_credit
```

- 如果能返回包含余额信息的 JSON，说明登录状态和环境配置正常。
- 如果这一步失败，不要继续提交生成任务，应先处理登录或配置问题。

### 推荐检查顺序

每次开始生成任务时，按这个顺序：

1. **参考图初始化**：检查 `assets/视频参考图/人物参考.*` 是否存在；不存在就停下来要求用户上传，保存好再继续。介绍产品的视频还要检查 `assets/视频参考图/产品参考/[产品名].*`。
2. `dreamina -h`
3. `dreamina user_credit`
4. 只有在 `user_credit` 失败时，才执行 `dreamina login`
5. 登录后再次执行 `dreamina user_credit`

## 视频任务命令

### 命令矩阵

| 命令                 | 功能       | 特点                 |
| ------------------ | -------- | ------------------ |
| `image2video`      | 单图生视频    | 最基础，支持最多模型版本       |
| `multimodal2video` | **全能参考（默认）** | 最强模式，支持图+视频+音频混合输入 |
| `multiframe2video` | 多图故事     | 2-20 张图连贯叙事         |
| `frames2video`     | 首尾帧生视频   | 给定开始和结束画面，AI 补中间    |

### 默认命令：multimodal2video（全能参考）

默认形态（只带人物参考，适用于绝大多数选题）：

```bash
dreamina multimodal2video \
  --image ./assets/人物参考.png \
  --prompt="$(cat prompt-7layer.txt)" \
  --model_version seedance2.0fast_vip \
  --duration=15 \
  --ratio=9:16 \
  --video_resolution=720p \
  --poll=30

# 提交后立即打开即梦网站查看进度
open "https://jimeng.jianying.com"
```

涉及产品介绍时（追加产品参考）：

```bash
dreamina multimodal2video \
  --image ./assets/人物参考.png \
  --image ./assets/产品参考.png \
  --prompt="$(cat prompt-7layer.txt)" \
  --model_version seedance2.0fast_vip \
  --duration=15 \
  --ratio=9:16 \
  --video_resolution=720p \
  --poll=30

open "https://jimeng.jianying.com"
```

适用：
- **所有视频任务的默认命令**，无论有没有视频/音频参考。
- **图片1（人物参考）默认必带**：第 1 张 `--image` 永远是人物参考。
- **图片2（产品参考）按需追加**：只有当视频内容明确介绍产品优点（结构、工艺、卖点、对比、测评等）时，才把产品参考作为第 2 张 `--image` 追加，并在 prompt 里同步加入"图片2"标记。
- 不带产品参考时，prompt 里**禁止出现"图片2"**，避免模型错绑。

参数表（来自即梦官方）：

| 参数                   | 类型            | 说明         | 可选值/范围                                                                  |
| -------------------- | ------------- | ---------- | ----------------------------------------------------------------------- |
| `--image`            | `stringArray` | 输入图片，可重复多次 | 最多 9 张，本地路径                                                               |
| `--video`            | `stringArray` | 输入视频，可重复多次 | 最多 3 个，本地路径                                                               |
| `--audio`            | `stringArray` | 输入音频，可重复多次 | 最多 3 个，2-15 秒                                                              |
| `--prompt`           | `string`      | 多模态编辑提示词   | 7 层结构整段，内联使用"图片1""图片2"绑定参考图                                                                      |
| `--duration`         | `int`         | 视频时长       | 4-15 秒，默认 5 秒                                                              |
| `--ratio`            | `string`      | 画面比例       | `1:1` `3:4` `16:9` `4:3` `9:16` `21:9`                                  |
| `--video_resolution` | `string`      | 分辨率        | `720p`                                                                  |
| `--model_version`    | `string`      | 模型版本       | `seedance2.0` `seedance2.0fast` `seedance2.0_vip` `seedance2.0fast_vip` |
| `--session`          | `int`         | 会话 ID       | 默认 0                                                                     |
| `--poll`             | `int`         | 提交后轮询等待秒数  | 0 为禁用                                                                    |

参考图绑定约定：
- `--image` 的传入顺序就是 prompt 里"图片N"的编号顺序。
- 图片1 = 人物参考（默认必带），图片2 = 产品参考（仅介绍产品时带）。
- 需要更多角色（场景参考、风格参考、第二人物等）时，按 图片3、图片4… 顺延，并在 prompt 里同步标注。
- 文件路径必须真实存在，否则命令会失败。

参考图存放路径约定：
- 人物参考：`assets/视频参考图/人物参考.png`（一份，全项目复用）
- 产品参考：`assets/视频参考图/产品参考/[产品名].png`（按产品分文件，如 `断桥铝型材.png` `系统窗.png`）
- **首次使用前必须完成"参考图初始化"**：如果 `assets/视频参考图/人物参考.*` 不存在，视频制作 Agent 必须停下来要求用户上传，再继续后续生成。详见 `视频制作/SKILL.md` 的 Step 0。

- `--images` 必须指向本地真实文件路径。

## 异步任务处理

### 使用 `--poll`

在提交命令中加入 `--poll=<秒数>`，例如：

```bash
--poll=30
```

行为：
- 工具提交任务后，会以每秒 1 次的频率轮询状态。
- 如果在等待时间内完成，终端直接输出最终结果。
- 如果等待超时，终端先返回 `querying` 状态和 `submit_id`。

### 查询任务结果

```bash
dreamina query_result --submit_id=<你的_submit_id>
```

直接下载到指定目录：

```bash
dreamina query_result --submit_id=<你的_submit_id> --download_dir=./downloads
```

### 查看历史任务

```bash
dreamina list_task
dreamina list_task --gen_status=success
dreamina list_task --submit_id=<你的_submit_id>
```

## Agent 执行约定

### 默认生成策略

- 所有视频任务默认走 `dreamina multimodal2video`。
- **参考图组合按内容性质决定**：
  - 默认只带 图片1（人物参考）。
  - 内容明确介绍产品优点（结构、工艺、卖点、对比、测评等）时，再追加 图片2（产品参考）。
  - 同一段拆分出的多个镜头，参考图组合保持一致。
- F1 15 秒短视频：默认整段生成 1 条视频任务，不先拆镜头。
- F2 90 秒案例拆解：按段落拆成多个 5 秒或 10 秒镜头，每个镜头都是一条 `multimodal2video` 任务。
- F3 30 秒情绪钩子：优先评估整段生成；效果不稳时再拆成 2-3 段。情绪钩子型一般不带 图片2。
- F4 图文长帖：统一使用 `text2image` 或 `image2image` 生成封面图和配图，不调用视频命令。

### 15 秒整段 Prompt 模板（7 层结构）

> **重要：传给 `--prompt` 的内容必须是 7 层结构整段，不要压缩成"主体+场景+动作+风格"的单段描述。**
> 文案写手输出的脚本里已经写好了 7 层结构 prompt，视频制作直接搬运即可。

7 层结构（缺一不可）：

| 层 | 作用 |
|----|------|
| 【风格锚定】 | 整体视觉调性 |
| 【光影色调】 | 光线、色温、画质 |
| 【运镜节奏】 | 镜头组合顺序与节奏 |
| 【人物/产品设定】 | 身份、服装、表情、语气、动作 |
| 【分镜脚本】 | 分镜号 + 画面 + 台词 + 字幕 + 音效 |
| 【场景环境】 | 拍摄空间与窗外内容 |
| 【整体氛围】 | 一句话定调 |

通用模板（图片1 默认必带；图片2 仅介绍产品时追加）：

```text
【风格锚定】[整体视觉调性]
【光影色调】[光线 + 色温 + 画质]
【运镜节奏】[镜头组合顺序和节奏]
【人物/产品设定】[身份] 图片1 ，[气质、表情、语气、动作]      # 介绍产品时改写为：[身份] 图片1 ，[与产品的关系/手持产品] 图片2 ，[表情、语气、动作]
【分镜脚本】15秒短视频
  0-3秒 | 镜头1 - [作用]
    画面：[具体画面构图]
    台词："[钩子台词]"
    字幕：[字幕样式]
    音效：[环境音/音效]
  3-7秒 | 镜头2 - [作用]
    画面：[具体画面构图]
    台词："[展开台词]"
    音效：[音效]
  7-11秒 | 镜头3 - [作用]
    画面：[具体画面构图]
    台词："[判断/对比台词]"
    字幕：[关键信息标注]
  11-15秒 | 镜头4 - [作用]
    画面：[人物看向镜头]
    台词："[收尾台词]"
    字幕：[互动字幕]
【场景环境】[拍摄空间 + 窗外内容]
【整体氛围】[一句话定调]
```

> "图片1""图片2"是参考图绑定标记，对应 `dreamina multimodal2video --image` 按顺序传入的本地文件。
> **图片1 在【人物/产品设定】里默认必出现一次**。
> **图片2 只在内容明确介绍产品优点时才出现**——通常放在【人物/产品设定】里，必要时也可在产品特写镜头的"画面"里追加。
> 不带图片2 的视频，prompt 里不要出现"图片2"标记。

写法要求：

- 只保留 1 个核心卖点，不把 3 个卖点塞进同一条 15 秒视频。
- 动作写成连续过程，不要写成 3 次跳切。
- 人物有台词时必须明确"看向镜头"。
- 优先写用户能感知的结果，例如"安静下来""不再西晒发烫""开窗更安心"。
- 少写抽象词，多写可拍到的动作、表情、环境变化。
- 如果是前后对比，也尽量放在同一场景里完成，不要跨太多空间。

适合的 15 秒内容类型：

- 场景演示型：先有问题，后有改善。
- 前后对比型：同一空间里出现明显变化。
- 单点说明型：只展示一个判断标准或一个动作。
- 情绪钩子型：先给情绪，再给一个结论。

示例（介绍产品类内容，含 图片1 + 图片2）：

```text
【风格锚定】苹果发布会式知识科普，专业干净，生活化真实感
【光影色调】南方回南天室内实景，自然光偏冷，窗框水珠反光清晰可见，湿度计屏幕微光，4K超清
【运镜节奏】中景交代潮湿环境→特写窗框水珠→手指划过密封条微距→湿度计数据特写→回到人物中景，节奏平稳克制
【人物/产品设定】门窗顾问 图片1 ，站在湿漉漉的窗边 图片2 ，表情从容不紧不慢，语气平实自信，语速中等，手自然指向窗框细节
【分镜脚本】15秒短视频
  0-3秒 | 镜头1 - 钩子
    画面：窗框和墙壁挂满水珠的回南天实景，人物站在窗边转头看镜头
    台词："窗框上挂满水珠，墙壁也开始'流泪'——回南天来了。"
    字幕：大字居中弹出"回南天"
    音效：环境雨声/潮湿氛围音
  3-7秒 | 镜头2 - 知识拆解
    画面：人物手指划过窗框密封条，微距展示角部接缝细节
    台词："很多人第一反应是买除湿机，但问题出在窗户。普通窗密封条角部断开，潮气直接钻进来。"
    字幕：标注"密封条角部断开"
    音效：指尖轻触窗框的细微声音
  7-11秒 | 镜头3 - 对比验证
    画面：切换到干燥的系统窗场景（或分屏对比），湿度计显示室内比室外低15%
    台词："好窗的三道密封，能把湿气挡在外面。窗框是干的，湿度低15%，这才是门槛。"
    字幕：标注"三道密封"+湿度数据"室内45% / 室外90%"
  11-15秒 | 镜头4 - 收尾引导
    画面：回到人物中景，自然看向镜头
    台词："回南天防潮，先别急着买除湿机，先看看你家窗户漏不漏气。你家最近也这样吗？"
    字幕：底部浮现"你家最近也这样吗？"
【场景环境】南方住宅室内，窗外隐约可见阴雨天气，室内窗框墙壁有明显回南天水汽凝结
【整体氛围】专业克制，像朋友在家里随口教你一招，信任感拉满，拒绝浮夸
```

  --video_resolution=720p \

```

  --duration=15 \
  --ratio=9:16 \
  --video_resolution=720p \
  --poll=30

# 提交后立即打开即梦网站查看进度
open "https://jimeng.jianying.com"
```

> 提示：`--image` 的传入顺序就是 prompt 里"图片N"的编号顺序——第 1 个 `--image` 永远绑定"图片1"，第 2 个 `--image` 永远绑定"图片2"。**确认两个顺序一一对应再提交**。

### 命令选择规则

- 15 秒默认整段视频：默认 1 条 `multimodal2video` 命令，带 图片1（人物）+ 图片2（产品）两张参考图。
- 完全没有任何参考图：才回退到 `text2video`，并在 prompt 里去掉"图片N"内联标记。
- 只有 1 张参考图、不需要绑定多角色：可使用 `multimodal2video` 单图，或回退到 `image2video`。
- 多帧故事/首尾帧需求：分别走 `multiframe2video` / `frames2video`。
- 只做封面和图文：使用 `text2image` / `image2image`。
- 需要查历史、补下载、补拉结果：使用 `list_task` / `query_result`。

### 结果落盘

- 生成结果默认下载到项目内 `output/videos/YYYY-MM-DD-[topic]/raw/`
- 成片候选、封面候选、任务 JSON、submit_id 记录都要保留，便于审核运营复核。

## 常见问题

### 登录后仍提示无权限或失败

先检查两项：

1. `~/.dreamina_cli/config.toml` 是否存在且内容正常。
2. `dreamina user_credit` 是否能成功返回信息。

如果 `user_credit` 失败，说明登录或环境配置异常，不要继续测试生成命令。

### 登录流程卡住

执行：

```bash
dreamina login --debug
```

把终端里的调试信息作为排障依据。

### 异步任务迟迟没有最终结果

- 优先用带 `--poll=30` 的命令提交。
- 超时后记录 `submit_id`，稍后执行：

```bash
dreamina query_result --submit_id=<你的_submit_id>
```

### 切换账号

```bash
dreamina relogin
```

### 清空本地登录信息

```bash
dreamina logout
```

注意：
- 该命令只清除 `credential.json` 中的登录凭证。
- 不会删除 `config.toml` 和 `tasks.db`。

## 本地文件说明

- `~/.dreamina_cli/config.toml`：环境配置文件。
- `~/.dreamina_cli/credential.json`：本地登录凭证。
- `~/.dreamina_cli/tasks.db`：任务提交记录数据库。
- `~/.dreamina_cli/logs/`：工具运行日志目录。
