# CTA 审核规则 (cta.md)

> 版本：v1.0 | 用途：审核运营审核结尾引导是否和内容目标匹配。

---

## CTA 类型

### A类：选择题型

适合利他型、评论互动型内容。

示例：
- 你家更在意安静、健康还是安全？
- 你家是高层、临街还是有娃家庭？

### B类：信息交换型

适合转化导向、咨询导向内容。

示例：
- 评论"选窗"，发你一份不同楼层的配置建议
- 评论你家的楼层+朝向，帮你判断先解决什么

### C类：经验共鸣型

适合传播型内容，引发用户分享经历。

示例：
- 你家装修时在窗户上踩过坑吗？
- 有没有住进去才发现窗户选错的经历？

### D类：下一条预告型

适合系列内容或 F3 情绪钩子，承接下一条内容。

示例：
- 下一条讲：怎么判断你家的窗该不该换
- 这个系列还有 2 期，关注不迷路

---

## 审核匹配规则

| 内容类型 | 推荐 CTA |
|---------|---------|
| 60%利他型（科普/避坑） | A类 或 C类 |
| 场景/案例型 | B类 |
| 产品/品牌型 | B类 |
| 30秒情绪钩子 | D类 |
| 系列内容 | D类 |

---

## 红线

- 同一条内容只能保留 1 个 CTA。
- 不得出现逼单、倒计时、名额稀缺等促销话术。
- 不得在评论区引导中放微信号、二维码、外链。
